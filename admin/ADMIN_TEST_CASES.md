# 管理后台测试用例

## 一、登录模块

### TC-LOGIN-001: 正确账号密码登录
- **前置条件**: 未登录状态
- **步骤**: 输入 admin / admin123，点击登录
- **预期**: 登录成功，跳转到仪表盘

### TC-LOGIN-002: 错误密码登录
- **前置条件**: 未登录状态
- **步骤**: 输入 admin / wrongpassword，点击登录
- **预期**: 显示"用户名或密码错误"

### TC-LOGIN-003: 空账号登录
- **前置条件**: 未登录状态
- **步骤**: 不输入用户名，点击登录
- **预期**: 表单验证提示"请输入用户名"

### TC-LOGIN-004: 登录状态保持
- **前置条件**: 已登录
- **步骤**: 刷新页面
- **预期**: 保持登录状态

### TC-LOGIN-005: 退出登录
- **前置条件**: 已登录
- **步骤**: 点击退出登录
- **预期**: 清除登录状态，跳转到登录页

## 三、活动管理模块

### TC-ACT-001: 活动列表展示
- **前置条件**: 已登录
- **步骤**: 进入活动管理页面
- **预期**: 显示活动列表，包含ID、名称、状态、奖励积分等

### TC-ACT-002: 创建活动 - 基本信息
- **前置条件**: 已登录
- **步骤**: 
  1. 点击"新建活动"
  2. 填写活动名称、描述
  3. 选择审核类型、参与频次
  4. 设置奖励积分
  5. 点击创建
- **预期**: 创建成功，返回列表页

### TC-ACT-003: 创建活动 - 表单字段配置
- **前置条件**: 已登录
- **步骤**: 
  1. 点击"新建活动"
  2. 添加多个表单字段（文本、图片、数字等）
  3. 设置字段必填属性
  4. 点击创建
- **预期**: 活动创建成功，表单配置保存

### TC-ACT-004: 创建活动 - 必填校验
- **前置条件**: 已登录
- **步骤**: 不填写活动名称，直接点击创建
- **预期**: 显示验证错误"请输入活动名称"

### TC-ACT-005: 编辑活动
- **前置条件**: 已有活动
- **步骤**: 
  1. 点击活动的"编辑"按钮
  2. 修改活动名称
  3. 点击保存
- **预期**: 修改成功，列表更新

### TC-ACT-006: 结束活动
- **前置条件**: 已有进行中的活动
- **步骤**: 点击活动的"结束"按钮
- **预期**: 活动状态变为"已结束"

### TC-ACT-007: 删除表单字段
- **前置条件**: 活动编辑页面，有多个表单字段
- **步骤**: 点击某个字段的"删除"按钮
- **预期**: 该字段被移除

### TC-ACT-008: 活动详情查看
- **前置条件**: 已有活动
- **步骤**: 点击活动名称或详情按钮
- **预期**: 显示活动完整信息

## 四、审核管理模块

### TC-SUB-001: 提交列表展示
- **前置条件**: 已登录
- **步骤**: 进入审核管理页面
- **预期**: 显示提交列表，默认筛选"待审核"

### TC-SUB-002: 状态筛选
- **前置条件**: 已登录
- **步骤**: 切换筛选条件（全部/待审核/已通过/已拒绝）
- **预期**: 列表按状态筛选

### TC-SUB-003: 查看提交详情
- **前置条件**: 有提交记录
- **步骤**: 点击"查看详情"
- **预期**: 弹窗显示提交数据和附件

### TC-SUB-004: 附件预览
- **前置条件**: 提交有附件
- **步骤**: 在详情弹窗中点击附件图片
- **预期**: 图片放大预览

### TC-SUB-005: 审核通过
- **前置条件**: 有待审核的提交
- **步骤**: 
  1. 点击"通过"按钮
  2. 输入发放积分
  3. 确认
- **预期**: 状态变为"已通过"，积分已发放

### TC-SUB-006: 审核拒绝
- **前置条件**: 有待审核的提交
- **步骤**: 
  1. 点击"拒绝"按钮
  2. 输入拒绝原因
  3. 确认
- **预期**: 状态变为"已拒绝"

### TC-SUB-007: 已处理记录不可重复审核
- **前置条件**: 有已审核的提交
- **步骤**: 查看已审核记录的操作列
- **预期**: 显示"已处理"，无审核按钮
